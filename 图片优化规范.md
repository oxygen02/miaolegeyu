# 喵了个鱼小程序图片优化规范

## 一、问题总结

### 当前问题
1. **按钮上跳**：重置按钮显示/隐藏导致布局跳动
2. **图文不匹配**：图片文件内容与名称不对应（如猪肚鸡显示羊蝎子图片）
3. **图片过大**：导致加载缓慢

---

## 二、按钮上跳问题（已修复）

**解决方案**：重置按钮始终显示，未选中时显示为禁用状态
- 选中状态：正常显示，可点击
- 未选中状态：半透明（opacity: 0.5），不可点击

---

## 三、图文不匹配问题（已修复）

**问题原因**：动态生成图片路径时使用了错误的分类索引

**修复内容**：
- 修正了 `goToSubCategory` 方法中的图片路径生成逻辑
- 现在优先使用数据文件中定义的 `sub.image` 路径
- 确保 `category_XX_YY.png` 与实际菜品名称对应

## 四、图片裁剪规格建议

### 1. 竖屏显示区域计算

根据代码分析：
```css
.full-swiper {
  height: calc(100vh - 420rpx);  /* 减去顶部和底部区域 */
}
```

**iPhone 14 Pro Max 示例**：
- 屏幕高度：932px
- 可用高度：932 - 420 = 512px
- 屏幕宽度：430px

**推荐图片比例**：9:16（竖屏视频比例）

### 2. 裁剪规格

#### 方案 A：统一规格（推荐）
| 项目 | 规格 |
|------|------|
| 分辨率 | 720 × 1280 px |
| 比例 | 9:16 |
| 格式 | WebP（优先）或 JPG |
| 质量 | 80% |
| 单张大小 | < 100KB |

#### 方案 B：响应式规格
| 设备类型 | 分辨率 | 文件命名后缀 |
|---------|--------|-------------|
| 标准 | 720 × 1280 | _m |
| 高清 | 1080 × 1920 | _l |
| 缩略图 | 360 × 640 | _s |

### 3. 批量处理命令（ImageMagick）

```bash
# 1. 裁剪为 9:16 比例，从中心截取
magick input.jpg -gravity center -crop 9:16 +repage -resize 720x1280 output.webp

# 2. 批量处理文件夹
for file in *.jpg; do
  magick "$file" -gravity center -crop 9:16 +repage -resize 720x1280 -quality 80 "../output/${file%.jpg}.webp"
done

# 3. 压缩已有图片
magick input.png -resize 720x1280 -quality 80 output.webp
```

### 4. 在线工具推荐

| 工具 | 网址 | 功能 |
|------|------|------|
| Squoosh | https://squoosh.app/ | 压缩、转WebP |
| TinyPNG | https://tinypng.com/ | 智能压缩 |
| iLoveIMG | https://www.iloveimg.com/zh-cn | 批量裁剪 |

### 5. 文件命名规范

```
cuisine_all/
├── category_01_00.png  # 川渝-大类封面
├── category_01_01.png  # 川渝-麻辣火锅
├── category_01_02.png  # 川渝-水煮鱼
├── category_01_03.png  # 川渝-回锅肉
├── category_01_04.png  # 川渝-宫保鸡丁
├── category_01_05.png  # 川渝-麻婆豆腐
├── category_02_00.png  # 湘赣-大类封面
└── ...

命名规则：category_{大类序号}_{细类序号}.png
- 大类序号：01-20
- 细类序号：00（封面）, 01-07（细类）
```
