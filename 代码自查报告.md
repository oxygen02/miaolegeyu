## 一、项目概述

### 1.1 小程序主要功能模块

| 模块 | 功能描述 | 核心页面/云函数 |
|------|---------|----------------|
| **首页** | 展示房间列表、创建房间入口 | index, create |
| **投票系统** | Mode A选饭店投票、Mode B偏好选择 | vote, submitVote, countVotes |
| **拼单功能** | 外卖拼单创建与参与 | create-group-order, group-order |
| **约饭功能** | 发起约饭活动 | fish-tank, dining-appointments |
| **店铺管理** | 店铺上传、详情展示 | shop-detail, upload-shop |
| **用户中心** | 个人资料、历史记录、收藏 | profile, settings, history |
| **美食发现** | 店铺推荐、美食探索 | food-discovery |

### 1.2 核心业务流程

```
创建房间流程:
用户选择模式(A/B/拼单) → 填写信息 → 调用createRoom云函数 → 生成roomId → 跳转到control页面

投票流程:
用户进入房间 → 加载房间数据(getRoom) → 展示投票界面 → 提交投票(submitVote) → 更新投票统计

拼单流程:
创建拼单 → 填写选项 → 发起拼单 → 分享邀请 → 参与者加入(joinRoom) → 达到条件后锁定(lockRoom)
```

### 1.3 数据库集合结构

| 集合名 | 用途 | 关键字段 |
|--------|------|---------|
| **rooms** | 房间信息 | roomId, title, mode, status, creatorOpenId, candidatePosters |
| **room_participants** | 房间参与者 | roomId, openid, joinedAt, status |
| **votes** | 投票记录 | roomId, openid, voteData, createTime |
| **group_orders** | 拼单信息 | roomId, shopName, platform, participants |
| **shops** | 店铺信息 | _id, name, location, images, ratings |
| **shop_favorites** | 店铺收藏 | shopId, openid, createTime |
| **dining_appointments** | 约饭活动 | shopId, initiatorOpenId, appointmentTime, deadline |
| **users** | 用户信息 | openid, nickName, avatarUrl |

## 三、高风险问题详细分析

### 3.1 数据库事务缺失（🔴 高风险）

**问题描述**:
在createRoom云函数中：
```javascript
// 1. 创建房间
await db.collection('rooms').add({ data: roomData });

// 2. 添加创建者为参与者
await db.collection('room_participants').add({
  data: {
    roomId,
    openid: OPENID,
    joinedAt: new Date(),
    status: 'joined'
  }
});
```

如果第一步成功，第二步失败，房间存在但创建者不在参与者列表中。

**修复建议**:
```javascript
await db.runTransaction(async transaction => {
  await transaction.collection('rooms').add({ data: roomData });
  await transaction.collection('room_participants').add({ data: participantData });
});
```

### 3.2 并发投票问题（🔴 高风险）

**问题描述**:
submitVote云函数中，多个用户同时投票时：
1. 用户A读取当前票数：10票
2. 用户B读取当前票数：10票
3. 用户A写入：11票
4. 用户B写入：11票（应该是12票）

**修复建议**:
使用原子操作：
```javascript
await db.collection('rooms').doc(roomId).update({
  data: {
    voteCount: _.inc(1)
  }
});
```

### 3.3 权限绕过风险（🟡 中风险）

**问题描述**:
getRoom云函数只根据roomId查询，不校验用户是否有权限查看：
```javascript
const room = await db.collection('rooms').where({ roomId }).get();
```

**修复建议**:
添加权限校验：
```javascript
// 检查用户是否在房间参与者列表中
const participant = await db.collection('room_participants')
  .where({ roomId, openid: OPENID })
  .get();

if (participant.data.length === 0 && room.status !== 'public') {
  return { code: 403, msg: '无权查看此房间' };
}
```
